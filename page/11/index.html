奖励值的设置

在正数代表奖励的情况下，0不惩罚，也是一种鼓励。

Reward(s,a)，奖励考虑当前状态和行为。

Trajectory（轨迹）:状态-动作-奖励 链。

return 是一个轨迹所有的奖励加起来的值。

如return=0+0+0+1=1

而在第9格时所有动作的收益将会一直加一，1+1+1+1+…+1=∞

为避免结果无穷大，引入折扣率，即discount γ∈[0,1]，

此时的dicount return=0+γ0+γ^20+γ^3\1+…+γ^n*1=γ^3*1\(1-γ)

γ趋近于0，短视。

Conflux伍鸣

为了提高效率，大体有以下这么几个思路：

1.依然采用中本聪共识，但调整协议参数。
通过调整出块时间和区块大小来提高效率。如莱特币(2.5 min/1MB block), BCH (10min/32MB block)。但有研究表明，无论这两个参数如何调整，提高吞吐率必然以降低安全性为代价，在第二部分，我将为大家分析这其中的原因。

2.基于中本聪共识的思想，使用 DAG优化POW。
这可以在不牺牲去中心化和安全性的前提下，大幅提高吞吐效率。
